## Sistema de Delivery

---

## Descrição

Este projeto consiste no desenvolvimento de um sistema de delivery.

---

## Requisitos Funcionais

Os requisitos funcionais do sistema:

- O sistema deve permitir o cadastro de usuários, incluindo clientes e administradores

- O sistema deve permitir o cadastro de produtos com nome, preço, descrição e imagem

- O sistema deve disponibilizar um carrinho de compras para os usuários

- O sistema deve permitir a realização de pedidos

- O sistema deve permitir o acompanhamento do status do pedido (recebido, em preparo, saiu para entrega e entregue)

- O sistema deve oferecer integração com formas de pagamento, como cartão, Pix e dinheiro

- O sistema deve armazenar e exibir o histórico de pedidos do usuário

- O sistema deve permitir a busca de produtos

- O sistema deve disponibilizar um painel administrativo para gerenciamento de produtos e pedidos

---

## Requisitos Não Funcionais

Os requisitos não funcionais do sistema:

- Desempenho: o sistema deve responder rapidamente, mesmo com múltiplos usuários simultâneos

- Segurança: os dados dos usuários devem ser protegidos por meio de autenticação e criptografia

- Responsividade: a interface deve se adaptar a diferentes dispositivos, como celulares e computadores

- Disponibilidade: o sistema deve estar disponível para uso 24 horas por dia

- Escalabilidade: o sistema deve suportar o aumento no número de usuários e pedidos

- Usabilidade: a interface deve ser simples e fácil de utilizar

- Manutenibilidade: o código deve ser organizado e permitir fácil manutenção e atualização
